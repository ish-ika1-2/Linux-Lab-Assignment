Question 4: File Access and I/O Investigation

1. Open Files Identified

Command: lsof

Observation:
The command displays all open files and the processes using them.

2. File Descriptor Observations

Command: ls -l /proc/$$/fd

Observation:
The command shows file descriptors associated with the current shell process.

3. Output Redirection

Command: echo "Test Output" > output.txt

Observation:
The output was redirected and stored in output.txt.

4. Error Redirection

Command: ls invalidfile 2> error.txt

Observation:
The error message was redirected and stored in error.txt.

5. Resource Limits

Command: ulimit -a

Observation:
The command displays system resource limits for the current user.

Explanation:

Linux uses file descriptors to manage input, output, and error streams. Output redirection (>) sends command output to a file, while error redirection (2>) stores error messages separately. Resource limits help control system usage and prevent excessive resource consumption.
